SMART LIBRARY ADMIN PLATFORM - DESIGN THEME PROMPT

Use this prompt to design, extend, or regenerate screens for the Smart Library
administration platform. The product is a French academic-library back-office
for administrators who manage users, monitor loans, review statistics, detect
abuse, and configure system rules.

The interface must feel like a serious operational admin console: calm,
academic, precise, trustworthy, and efficient. It should not feel like a
marketing landing page, decorative portfolio, or playful consumer app. The
first priority is scanability and decision-making clarity.


1. PRODUCT PERSONALITY

Design tone:
- Quietly premium.
- Academic and institutional.
- Administrative, structured, and fast to scan.
- Modern but restrained.
- Confident use of whitespace, borders, and typographic hierarchy.

User emotion:
- The administrator should feel in control.
- Risk states should be obvious without making the whole interface alarming.
- Settings should feel stable and official.
- Tables should feel trustworthy and readable.

Avoid:
- Large marketing hero sections.
- Decorative blobs, orbs, abstract SVG backgrounds, and loud gradients.
- Overly rounded, bubbly UI.
- Excessive shadows.
- Purple/blue SaaS gradients.
- Dense clutter without spacing.


2. CORE LAYOUT

Global shell:
- Fixed left sidebar, width: 260px.
- Fixed topbar, height: 64px.
- Main content starts after sidebar and topbar.
- Body background: #f9f9f9.
- Surface/card background: #ffffff.
- Main content padding: 24px desktop, 16px on mobile.

Sidebar:
- Background: #f9f9f9.
- Border-right: 1px solid #c4c7c7.
- Brand block at top with local logo image.
- Navigation items stacked vertically.
- Active nav item has a left border in gold and a muted surface background.
- Icons use Material Symbols Outlined.
- Labels are French.

Topbar:
- White surface.
- Bottom border: #e5e5e5.
- Left: "Portail Admin".
- Center/right: search input.
- Right: notifications, history, profile avatar.
- Profile avatar uses local asset: assets/profile.png.

Responsive behavior:
- Hide sidebar below tablet width.
- Main content should fill full width when sidebar is hidden.
- Convert multi-column grids to one column on smaller screens.
- Never let text overlap or overflow its container.


3. COLOR SYSTEM

Primary text:
- Main ink: #1c1b1b.
- Strong primary: #181919.
- Secondary text: #5d5f5f.
- Muted text: #747878.

Surfaces:
- Page background: #f9f9f9.
- Primary surface: #ffffff.
- Soft surface: #f7f3f2.
- Muted surface: #ebe7e7.
- Strong muted surface: #e5e2e1.

Borders:
- Standard border: #c4c7c7.
- Soft border: #e5e5e5.

Gold accent:
- Main gold: #d4af37.
- Hover gold: #cca830.
- Dark gold text: #735c00.
- Pale gold: #ffe088.

Danger:
- Red: #ba1a1a.
- Dark red: #93000a.
- Soft red: #ffdad6.

Success:
- Green: #2e7d32.
- Soft green: #e8f5e9.

Warning:
- Orange: #e65100.
- Soft orange: #fff3e0.

Use color semantics carefully:
- Gold: primary action, accent, monetary amounts, warning-medium category.
- Red: destructive action, blocked state, critical abuse risk.
- Green: safe/active/success state.
- Neutral black/dark gray: table headers, serious admin controls.


4. TYPOGRAPHY

Font:
- Inter for everything.

Global body:
- 16px base.
- Line-height around 1.5.

Page title:
- Around 32px.
- Weight 600-700.
- Color #181919.
- Letter spacing 0.

Section/card title:
- Around 17px.
- Weight 700-800.

Labels:
- 12px to 14px.
- Weight 600-800 depending on importance.
- Often uppercase only in table headers or compact field labels.

Numbers/KPIs:
- Strong and large.
- Use 24px-32px depending on card size.
- Avoid oversized dashboard numbers that break compact admin feel.

Text rules:
- No negative letter spacing.
- Do not scale font size with viewport width.
- Use shorter French labels where possible.
- Keep table labels concise.


5. SHARED COMPONENT STYLE

Cards:
- Border radius: 8px maximum.
- Border: 1px solid #e5e5e5 or #c4c7c7.
- Background: #ffffff.
- Shadows should be very subtle and only for sticky/floating bars.
- Do not nest decorative cards inside large decorative cards.

Buttons:
- Primary button:
  - Background #d4af37.
  - Text #181919.
  - Border radius 8px.
  - Icon + label when command is important.
  - Hover #cca830.
- Secondary button:
  - White/transparent surface.
  - Border #c4c7c7.
  - Text #181919.
- Danger button:
  - Background #ba1a1a.
  - Text white.
  - Hover #93000a.
- Icon buttons:
  - Circular or small square.
  - Material Symbols icon.
  - Hover surface #f7f3f2.

Inputs/selects/textareas:
- Border #c4c7c7.
- Radius 6px.
- Padding around 10px-12px.
- Focus border #d4af37.
- Background white.

Toggles:
- Width 44px, height 24px.
- Off: #c4c7c7.
- On: #735c00.
- Thumb white, 18px.

Tables:
- Full width.
- Header row dark #2d2d2d with white text.
- Header text uppercase, 12px, bold.
- Body rows white with soft horizontal dividers.
- Hover: #f7f3f2.
- Use nowrap only when table has horizontal scroll.

Status pills:
- Rounded full.
- 12px-13px, bold.
- Active: neutral/gold dot.
- Blocked/high risk: red background/text.
- Medium risk: pale gold/orange.
- Low risk: pale green/green.


6. IMAGE ASSETS

Use local assets:
- Logo: assets/logo.png.
- Profile picture: assets/profile.png.

Logo placement:
- Sidebar brand logo, 38px by 38px, object-fit contain.
- Settings profile panel may use a larger 86px logo preview.

Avatar:
- Topbar avatar is circular, 34px by 34px.
- object-fit: cover.


7. PAGE: TABLEAU DE BORD

Purpose:
- Provide immediate operational overview.

Design:
- KPI cards in a grid.
- Audit log table.
- Risk level panel.
- Use compact cards and clear hierarchy.

Suggested content:
- Utilisateurs Actifs.
- Emprunts en Cours.
- Retours en Retard.
- Journal d'Audit Recent.
- Niveaux de Risque.

Visual rules:
- Avoid decorative hero.
- Keep dashboard dense but breathable.
- Critical data should be visible without scrolling on desktop.


8. PAGE: GESTION DES UTILISATEURS

Purpose:
- Manage user records, statuses, risk levels, and sanctions.

Design:
- Page title plus description.
- Toolbar with role/status filters and "Nouvel utilisateur" action.
- Main table is the hero of the page.
- Pagination footer.

Table columns:
- ID.
- Nom.
- Role.
- Statut.
- Niveau d'Alerte.
- Retards.
- Penalites.
- Actions.

Action buttons:
- visibility.
- edit.
- block or lock_open.
- gavel.

Pagination:
- Show compact page buttons.
- Last visible page value should be 18.


9. PAGE: STATISTIQUES

Purpose:
- Show analytical trends without changing into a marketing dashboard.

Current visual direction:
- Preserve the elegant curve chart.
- Use compact KPI cards.
- Keep "Statut des Retours" donut card clean and simple.

Top KPI values:
- Livres endommages: 12.
- Penalites appliquees: 2.
- Taux de reussite (reservations): 94.5%.
- Emprunts actifs: 65.

Curve panel:
- Title: Tendances d'Emprunt (6 Derniers Mois).
- Use gold line/area chart.
- Soft dotted grid lines.
- Chart fill fades downward.
- Points are small and clean.
- Axis labels: Jan, Fev, Mar, Avr, Mai, Juin.

Return status card:
- Title: Statut des Retours.
- Donut with black/dark gray 82% and gold 18%.
- Center text: 82%, A l'heure.
- Legend:
  - Retours a l'heure: 82%.
  - Retours en retard: 18%.
- Donut should be large enough to fill the card space.

Table:
- Keep existing table style.
- Only values/content may change unless asked.


10. PAGE: CENTRE DE DETECTION DES ABUS

Purpose:
- Operational monitoring page for abuse, sanctions, alerts, and quick restrictions.

Design philosophy:
- This page should feel sharper and more urgent than the others, but still controlled.
- Use red only for critical risk and destructive actions.
- Use gold for medium risk, warning, and monetary values.

Layout:
- Header with title, description, and "Politique de Sanction" button.
- Summary row with 4 compact cards:
  - Alertes critiques: 2.
  - Alertes actives: 4.
  - Restrictions en attente: 1.
  - Penalites ouvertes: 60.50euro.
- Two independent columns:
  - Left/main: Utilisateurs a Haut Risque, then Historique des Penalites.
  - Right/side: Alertes Comportementales, then Action Rapide.
- The penalty table must sit close under the high-risk user card, with only a small gap.

High-risk user cards:
- Two cards side by side on desktop.
- Ayoub El Idrissi:
  - ID: 991-K.
  - Label: Score: High.
  - Metrics: 4 ouvrages, 30j+ retards.
  - Motif: Retards repetes (> 30 jours) sur ouvrages de reserve (x4).
  - Suspendre button must be red.
- Adam Bennis:
  - ID: 112-C7.
  - Label: Score: Medium.
  - Metrics: 1 incident, 1 ouvrage.
  - Motif: Signalement d'ouvrages endommages (annotations au stylo).
  - Avertir button uses gold.

Penalty table:
- Monetary amounts such as 15.00 euro and 45.50 euro use the same soft gold as the Avertir button (#d4af37).
- Suspension duration "3 Mois" uses red.

Alerts:
- Keep alert feed compact.
- Topline contains alert title and timestamp.
- Severe alerts use red border.
- Warning alerts use gold border.

Quick action:
- Form fields: ID Utilisateur, Type de restriction, Motif / Justification.
- Default restriction option: Suspension temporaire (Emprunts).
- Final button is red: Appliquer la restriction.
- Include audit note below the button.


11. PAGE: PARAMETRES

Purpose:
- Configure system identity, loan rules, notifications, and security.

Design direction:
- This is the most polished settings dashboard.
- It should feel official and premium.
- Use a settings overview row, then structured panels.

Top overview:
- Three boxes filling the row evenly:
  - Pret standard: 15 jours.
  - Alertes retard: Actives.
  - Securite: 2FA desactivee.
- Do not include an "Identite / Validee" overview box.

Panels:
- Profil de l'Etablissement:
  - Strong panel header.
  - Logo console on the left.
  - Form fields on the right.
  - Informational note below fields.
- Regles de Pret:
  - Four rule cards:
    - Duree de pret: 15 jours.
    - Livres max: 5 par utilisateur.
    - Penalite: 0.50euro par jour.
    - Delai de grace: 2 jours.
  - Each card contains a visible value and editable input.
- Notifications:
  - Option rows with toggles.
  - Alertes par email pour retards: on.
  - Newsletter mensuelle: off.
  - Resume hebdomadaire admin: on.
- Securite & Acces:
  - Expiration de session select.
  - 2FA warning status.
  - Changer le mot de passe button.

Save bar:
- Sticky near bottom.
- White surface, border, subtle shadow.
- Text: Derniere modification le 24 Octobre 2023 a 14:30.
- Buttons: Reinitialiser, Sauvegarder les modifications.


12. MOTION AND INTERACTION

Keep interactions simple:
- Hover background changes.
- Active buttons translate down by 1px.
- Toggles switch knob position.
- Avoid large animations.
- No decorative motion.

Focus states:
- Use gold focus ring:
  box-shadow: 0 0 0 3px rgba(212, 175, 55, 0.22).


13. IMPLEMENTATION RULES

Use:
- Plain HTML, CSS, and small vanilla JavaScript.
- One main file: index.html.
- One stylesheet: styles.css.
- One script: script.js.
- Hash-based navigation:
  - #dashboard
  - #users
  - #statistics
  - #abuse
  - #settings

Do not require a server:
- The app must work from file://.

Keep text French in the UI.
Use HTML entities or ASCII-safe text if encoding is uncertain.


14. QUALITY BAR

A screen is successful if:
- The main task is obvious within 2 seconds.
- The active sidebar section is clear.
- Every card has a reason to exist.
- Empty space feels intentional.
- Tables are readable.
- Risk and action severity are visually correct.
- The page looks coherent with the rest of the admin platform.

A screen is not successful if:
- It feels like a landing page.
- It uses decorative gradients or bokeh.
- It relies on oversized cards with little data.
- It hides important admin actions.
- It changes the design language page by page.
- It introduces colors outside the approved palette without a reason.
